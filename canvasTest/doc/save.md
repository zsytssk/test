## 代码的结构

-   api --> data --> render (canvas | webgl)

    -   相互独立

-   api: 是独立 抽出给外层调用, 面向对象的方式组织

    -   用于填充数据...
    -   方便外层调用
    -   最好兼容 laya 接口

-   dom: 绘制 ui 所需的数据 - dop

    -   相当于虚拟 dom
    -   层级结构 各层各种数据
    -   最精简的结构, 最快的渲染

-   render: (canvas| webgl) -- 函数式

    -   调用 canvas webgl 渲染 页面
